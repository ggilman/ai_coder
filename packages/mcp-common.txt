# ============================================================
# mcp-common.txt — core MCP servers registered for all MCP-capable
# agents (Claude, Gemini, OpenCode). Not used for Aider.
#
# Keep this list SMALL. Every registered server's tool schemas are
# injected into the model's context on every request — each extra
# server slows prompt processing and makes small local models worse
# at picking the right tool. Servers that are useful but not
# essential belong in mcp-extra.txt (enabled via ai-coder --setup).
#
# Format (pipe-delimited, one server per line):
#   npm-package | server-key | command | arg1 arg2 ... | ENV_VAR1,ENV_VAR2 | net
#
# The optional 5th field lists env var NAMES (from the host shell) whose
# values are expanded and embedded in the generated MCP config at startup.
#
# The optional 6th field (net): set to "online" to skip the server when
# network isolation is active (ai-coder --setup → isolate containers).
# Leave blank for servers that work fully offline (local file I/O only).
#
# Use {workspace} as a placeholder for the container workspace path.
# Lines starting with # are ignored. Blank lines are ignored.
# ============================================================

# Git tools: status, diff, add, commit — used to navigate merge conflicts
pip:mcp-server-git                          | git        | mcp-server-git                   | --repository {workspace}

# Filesystem tools: reliable whole-file read/write bypassing str_replace fragility
@modelcontextprotocol/server-filesystem       | filesystem | mcp-server-filesystem            | {workspace}

# Shell command execution — run cmake, make, ctest, bash scripts directly from the agent.
# Configured via env vars. ALLOWED_COMMANDS=all and ALLOWED_FLAGS=all permit any command/flag.
# ALLOW_SHELL_OPERATORS=true enables &&, ||, |, > etc.
pip:cli-mcp-server                          | shell      | cli-mcp-server                   | | ALLOWED_DIR={workspace},ALLOWED_COMMANDS=all,ALLOWED_FLAGS=all,ALLOW_SHELL_OPERATORS=true,COMMAND_TIMEOUT=120
